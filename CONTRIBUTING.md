# Contributing Guide

We welcome contributions to OSS Forge. Here's how you can participate:

## Issues

- Bug reports and improvement requests should be submitted to [Issues](../../issues)
- Provide clear descriptions and reproducible steps
- Use appropriate labels for categorization

## Pull Requests

- Create PRs for new templates and improvement proposals
- Run lint/format checks before submitting PRs
- Keep PRs focused and atomic
- Link to related issues when applicable

## Development Rules

- Use schema (`scaffold.schema.json`) as the single source of truth
- PRs must pass schema consistency CI checks
- Generated outputs must pass validation tests
- Do not include secrets or sensitive information in examples

## Code of Conduct

- Please follow [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md) at all times
- Be respectful and constructive in all interactions
- Focus on technical discussions and collaboration

## Getting Started

1. Fork the repository
2. Create a feature branch
3. Make your changes
4. Test your changes
5. Submit a pull request

Thank you for contributing to OSS Forge!
